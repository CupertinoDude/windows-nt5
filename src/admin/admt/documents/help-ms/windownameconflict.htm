Naming Conflicts

Specifies how the Active Directory Migration Tool handles accounts in the target domain that have the same name as a copied account in the source domain. The options are defined as follows:

Ignore conflicting accounts and don't migrate

Leaves the account in the target domain unchanged.

Replace conflicting accounts

Changes properties of existing accounts in the target domain to match the properties of the account with the same name in the source domain.

Remove existing user rights

Ensures the account in the target domain does not have more user rights than those granted to the account with the same name in the source domain. This check box is only available when Replace conflicting accounts is selected.

Remove existing members of groups being replaced

Ensures the members of the migrated groups in the target domain are the same as the members of the associated groups in the source domain. This check box is only available when Replace conflicting accounts is selected.

Rename conflicting accounts by adding the following

Adds the specified prefix or suffix to the name of the migrated account in the target domain to avoid the conflict with the other account with the same name in the target domain. The original account with the same name in the target domain remains unchanged.
