64-Bit Key Support in WMI

 

Rev 1.00, 19-Jul-99, raymcc

This memo concerns the support for 64-bit keys in WMI Nova M3.   Further changes/improvements are expected in WMI Quasar and WMI 64-bit.

If a CIM 64-bit type is used for a class,

class MyBig
{
    [key] sint64 k;
    sint64 p;
};

instance of MyBig
{
    k = 2;
    p = 3;
};

The object path to an instance based on a 64-bit key is always enclosed in quotes.     The path to the above instance is

        MyBig = "2"

As long as the 32-bit range is not exceeded, the path may omit the quotes and use hex or decimal notation, as for 32-bit paths. 

Once the range of the integer exceeds 0x7FFFFFFF (for signed types) or 0x80000000 (for unsigned types), or 32 bits, the quotes must be used. Further, the only valid path is the one in the __RELPATH or __PATH properties of the instance.     Thus, hexadecimal notation for the equivalent value, for example, is not supported.

Likewise, if the instance key was recorded as a negative number, then that original number is the one that must be used to retrieve it.

Query semantics are unaffected and behave as expected.  This behavior only affects the object path and GetObject/GetObjectAsync operations.
