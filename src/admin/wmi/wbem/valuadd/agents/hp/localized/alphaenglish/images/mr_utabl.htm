Understanding Message Reader's tabular display

The default Message Reader tabular display shows information about all messages received from machines in your local domain and subnet that have ManageX installed. Each column of the display presents information as described below.

CAUTION:
You must click on the Message Reader at least once in each ManageX session to activate its dynamically updating system. You will receive no messages until you have done this.

The meanings of the columns are as follows:

  • Icon/Type—Indication of the type of message. received (such as Information, Warning, Error, etc.)
  • Time—The date and local time this message was received. Format and display options are available in Properties.
  • Source—The source or origin of this message. Although application name is commonly used, this value is user-defined and may be a subsystem name (for example, SQL Server) or any meaningful name.
  • Category—The type of event that generated this message, for example, START, STOP, or INSTALL.
  • Event ID—The identifying number of the event.

    NOTE: In the full ManageX product, Event IDs generated by ManageX policies are listed in a searchable format within the online Help.

  • Computer—The name of the machine that generated the message.
  • Acknowledged—Yes/No indication of message acknowledgment. A message can be marked as having been read and processed, or acknowledged, from the message's shortcut menu.
