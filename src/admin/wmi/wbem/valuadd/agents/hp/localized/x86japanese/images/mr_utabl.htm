メッセージ リーダの表形式表示について

メッセージ リーダのデフォルトの表には、Manage がインストールされているローカル ドメインとサブネットのマシンが受信したすべてのメッセージに関する情報が表示されます。各列には下記のような情報が表示されます。

注意:
動的更新システムを有効にするには、各 ManageX セッションでメッセージ リーダを少なくとも 1 度クリックする必要があります。これを行うまでは、メッセージは受信されません。

表は MMC の結果ウィンドウ (右側) に次のように表示されます。

各列の意味は次のとおりです。

  • アイコン/タイプ - 受信したメッセージの種類 (情報、警告、エラーなど)。
  • 時刻 - このメッセージを受信した日付とローカル時間。形式と表示のオプションは [プロパティ] で選択できます。
  • 送信元 - このメッセージの送信元またはソース。通常はアプリケーション名が使用されますが、この値はユーザーが定義するので、サブシステム名 (たとえば SQL Server) や意味のある他の名前の場合もあります。
  • カテゴリ - このメッセージを生成したイベントの種類。たとえば開始、停止、インストールなど。
  • イベント ID - イベントの識別番号。

    • 注: ManageX のフルバージョンでは、ManageX ポリシーが生成するイベント ID はオンライン ヘルプ内から検索可能な形式で一覧表示されます。

  • コンピュータ - メッセージを生成したマシンの名前
  • 確認済み - メッセージの確認の有無。メッセージを読んで処理した (確認済み) ことを示すマークは、メッセージのショートカット メニューで挿入できます。
