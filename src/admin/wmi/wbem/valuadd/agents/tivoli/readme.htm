T M A
Tivoli Management Agent

In 1998, Tivoli Systems Inc. extended its market-leading enterprise IT management product with a new technology that enables customers to reach their management goals with significantly less work — One Touch Management™. This same technology has also created an industry movement toward manageability, where hardware and software manufacturers can embed a lightweight management agent called the Tivoli Management Agent (TMA) to give their customers out-of-the-box management.

The TMA is a component of the Tivoli three-tier management hierarchy. It is a lightweight agent, designed to accept new functionality pushed from the second tier (gateway) to provide an unlimited range of management functions. This same push technology allows the TMA to be self-managing and self-updating, eliminating the need to physically visit a system once the agent is installed.

The TMA allows Tivoli to manage any device-from large mainframes to the newest PDAs and pervasive computing devices.

The TMA allows Tivoli to provide unprecedented scalability, supporting hundreds of thousands of devices under a single management server-a region. By connecting regions, you can manage an unlimited number of devices in your IT environment.

The agent's ability to automatically update its internal software and receive new functionality from a gateway means that once a managed resource (or endpoint) has the TMA installed, you never need to visit that endpoint again. This is One Touch Management-and it is unique in the industry.

Perhaps even more importantly, the advent of the TMA has brought about a revolution not only of management, but also of manageability. For vendors of hardware and software products, providing manageable systems gives them a competitive advantage. These products that embed the TMA technology provide the customer with out-of-the-box management, or No Touch Management-products such as Microsoft Windows 2000, 3Com Network Interface cards, every Intel Wired for Management computer, and every IBM computer and enterprise software package. Currently millions of TMA-enabled systems ship every month.

As Tivoli moves into the pervasive computing industry, managing products from mobile phones, storage area network devices, and even non-standard devices such as power generators and aircraft engines, the customer requirements do not change: manageability is a key value proposition to the consumers of these devices. For customers, buying a management-ready product allows them to focus on using a product, rather than making a product work.

For the vendors, solving management problems can be difficult. Tivoli provides an extensible management framework that is unique in the industry. By leveraging the Tivoli solution, vendors are able to focus on their products' functions, rather than management issues.

To install the Tivoli agent:

  1. Double click on setup.exe.
  2. Click next at the Welcome panel
  3. Choose to install the Tivoli Management Agent at the Endpoint Installation Options Screen. It will have a Destination Directory "C:\Program Files\Tivoli\lcf".
  4. Click the Browse Button if you want to change the Destination Directory for installing the Tivoli Endpoint.
  5. Enter in Name and Password for Remote User File Access
  6. Enter your advanced configuration options and finish the installation
  7. Check the box if you wish to see the Readme in the Setup Complete screen.
  8. Click on the finish button to dismiss this setup program.

To uninstall the Tivoli agent:

  1. Go to the directory that you have TMA installed and look for uninst.bat. (typically this is in the "C:\Program Files\Tivoli\lcf" directory)
  2. Run the uninst.bat program.
  3. A dialog panel will come up with a Confirm File Deletion Panel, click Yes.
  4. Click ok at the Remove Programs from your Computer screen.
  5. Follow directions to cleanup leftover files.
  6. When the uninst is done you should have no files/directory for your endpoint and no Tivoli Endpoint process running in services. If you see a Tivoli Endpoint in the Services Panel that did not get removed through uninst, the following command will stop the service and remove the service from the registry:
    <path>\lcfd.exe -r <hostname>
    <path> is where you have Tivoli endpoint installed.
