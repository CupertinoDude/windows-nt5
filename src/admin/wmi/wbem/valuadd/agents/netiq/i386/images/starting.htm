Starting and stopping Knowledge Script jobs

After you have installed the AppManager WMI Agent on the computer you want to manage, you can start and stop Knowledge Script jobs to begin collecting data and generating events.

The exact steps you follow to start and stop jobs depend on the type of WMI browser you are using. Here is a general example:

  1. Connect to the NetIQ namespace (<root>\netiq).

  2. Select the NetIQ_LRJob class.

  3. Enumerate the instances to display a list of all jobs.

  4. Double-click the row for the job you want to start (this zooms in on an individual instance).

  5. Select the status row, and then click the value cell on the status row.

  6. To start a job, type in the word "Running"; to stop a job, type the word "Stopped".

  7. If you want the Knowledge Script to generate an event when one of the preset thresholds is exceeded, set the CollectEvents property to Y. To suppress events, set the CollectEvents property to N.

  8. If you want the Knowledge Script to collect performance data, set the CollectData property to Y. If you do not want to collect data, set CollectData to N.

  9. Save your changes.
