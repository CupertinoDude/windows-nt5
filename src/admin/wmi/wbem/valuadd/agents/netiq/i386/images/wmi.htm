About the AppManager WMI Agent

The AppManager WMI Agent provides a taste of AppManager's most useful and productive management and monitoring features for Windows 2000 servers and workstations. We have simplified configuration by choosing the most common default settings. For example, when you are using the full AppManager product to monitor Windows 2000 or Windows NT, you can specify a memory usage threshold; when memory usage exceeds this threshold, AppManager generates an event. In the AppManager WMI Agent, we have selected a memory usage threshold for you.

To use the WMI AppManager Agent:

  • Install the AppManager WMI Agent on every computer that you want to manage.
    • The Agent runs on the managed computer as the NetIQWMI service; its service description is NetIQ WMI Agent.
  • Use a WMI-enabled browser to connect to the managed computer.

Once connected to the managed computer, you can run Knowledge Script jobs on that computer. When you run a job, you have the option to:

  • Generate an event when a performance threshold is exceeded.
  • Collect data that can be used for performance analysis and reporting.

From the WMI browser, you interact with the managed computer using three classes that are registered in the NetIQ schema:

Use this class:

To:

NetIQ_LRJob

Start and stop jobs on the managed computer, and browse job status.

NetIQ_LREvent

Browse any events generated by jobs running on the managed computer.

NetIQ_LRGraph

Browse any data collected on the managed computer.
