Service Property Sheet

Use this property sheet to enable HTTP compression or to designate one Web site that can be administered programmatically by applications that use the IIS 3.0 administration interfaces.

Default Web Site

Select the one Web site that the IIS 3.0 administration interfaces will have access to.

HTTP Compression Configuration

Use HTTP compression to compress files before they are sent to compression-enabled clients. Once a file has been requested, the compressed copy is stored in a cache directory and this compressed file is sent when further requests are received. Files are not compressed prior to receiving a request for them.

Select Compress static files to compress static HTML files.

Select both Compress static files and Compress application files to compress application files.

Type the name and path of the directory in which you want to cache the compressed files in the Temporary folder text box. You can also click Browse to locate the directory. The directory must be on an NTFS partition on the local hard disk and should not be shared or marked for compression.

You can limit the size of the cache directory by typing a size in the Limit to text box.
