Configuring the Server for Applications

This section explains how to configure the server to run Web-based applications by using the IIS snap-in.

This section contains:

  • About Your Server and Applications: Describes considerations when configuring your server to run applications.
  • Application Pools: Describes configuring the server to pool application processes.
  • Application Pool Queue Length Limits: Describes setting the queue length limits for application pools.
  • Configuring Standard and Dedicated Application Mode: Describes configuring the modes of operation and the ramifications of each.
  • Switching between Standard and Dedicated Application Mode: Describes how to change the modes of operation and the ramifications of each.
  • Web Gardens: Describes how to configure multiple application pools running under one process.
  • Recyclable Worker Processess: Explains configuring IIS to periodically restart worker processes in an application pool.
  • Configurable Worker Process Identity: Describes the security considerations when configuring the identity of a worker process.
  • Orphaned Worker Processes: Describes how a misbehaving process can be configured to replace itself.
  • Health Monitoring: Describes how IIS monitors the health of each worker process.
  • Process Demand Start: Explains configuring a process to start on demand when the first request for a URL is received.
  • Idle Timeout: Describes the resource management feature idle process timeout.
  • Processor Affinity: Describes the concept of process/processor affinity.
  • Rapid Fail Protection: Explains how IIS can protect computer resources when the requests to a process begin to fail rapidly.
  • Process Recycling: Explains the methodology for recycling applications, configurable events, programming, and setup considerations.

This section does not contain information on configuring applications.
