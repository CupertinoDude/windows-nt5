Restoring the Metabase Manually using History Files

This section will contain metabase rollback scenarios that describe how to restore the metabase manually from history files.

Rollback Scenario 1
This scenario will discuss using the UI to rollback the metabse, and also manually stopping IIS, copying the old file(s) and re-starting IIS. It will be stressed that these are the recommended methods. There will also be a discussion here about copying the corresponding MBSchema.XML file over as well.

Rollback Scenario 2
This scenario will show the things that can happen (go wrong) when you copy an old metabase file over metabase.xml while IIS is running.

  • The corresponding history file may not exist
  • If the corresponding history file does exist, only the changes will be pushed to the in-memory metabase.

It will be stressed that rollback scenario 2 illustrates an improper practice, and will likely result in un-expected results. It will also be emphasized that Rollback scenario 1 should be used instead.
