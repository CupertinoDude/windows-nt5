Configuring Applications

This section explains how to configure a Web-based application by using the IIS snap-in.

This section contains:

  • About Applications: Defines Web-based applications and explains how your site's directory structure is related to the definition of an application.
  • Creating Applications: Setting up a directory as the starting point, or root, of an application.
  • Setting Application Mappings: Associating a file name extension with the program that processes those files.
  • Isolating Applications: Running an application in a separate memory space from the Web server memory space.
  • Caching Applications: Setting CGI script timeouts, caching ISAPI DLLs, and changing the caching of ASP pages.
  • Stopping Isolated Applications: Stopping an isolated application.
  • Configuring ASP Applications: Setting properties for ASP applications.
  • Enabling ASP Debugging: Enabling debugging on the Web server so that you can check and correct ASP pages.
  • Configuring CGI Applications: Installing and configuring CGI applications on the Web server.
  • Installing ISAPI Filters: Explains what ISAPI filters are and how to associate them with a Web server event.
  • Process Recycling: Explains the methodology for recycling applications, configurable events, programming, and setup considerations.

This section does not contain information on:

  • Writing ASP applications; see Active Server Pages Guide.
  • Using certificates; see Certificates.
