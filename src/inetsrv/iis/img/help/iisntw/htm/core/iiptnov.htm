Performance Tuning

Internet Information Services 5.1 is limited to ten simultaneous client connections. Performance tuning for ten simultaneous client connections is typically not necessary. For this reason, the following information is intended for audiences using Windows 2000 Server and Windows 2000 DataCenter or later, products capable of handling several thousand simultaneous client connections. If you are interested in performance tuning IIS 5.1, you should also consider the performance gains of creating applications with Active Server Pages (ASP). See the ASP Guide for more information.

This section explains how to select and use Windows XP tools to monitor both server and site performance, and how to adjust a number of settings to increase overall performance. IIS offers access to performance settings through two methods. The first method is to use the IIS snap-in. The second method is to make programmatic changes directly to settings in the metabase. This section only describes changing performance settings by using the IIS snap-in.

This section includes:

  • About Performance Tuning: Conceptual information describing performance issues.
  • Network Capacity: Planning network capacity, and maximizing connection performance.

This section does not include information on:

  • Programmatic administration; see Administering IIS Programmatically.
  • Performance effects of site content choices; see Configuring Applications.
  • Tracking Web site traffic; see Logging Site Activity.
