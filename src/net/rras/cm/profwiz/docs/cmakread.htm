Welcome to the Connection Manager Administration Kit (CMAK).

Please print and read all information contained in this Readme file and the Connection Manager Readme (Readme.txt). After installing the Connection Manager Administration Kit, please read the Connection Manager Administration Kit Guide. These documents contain critical information for ensuring proper installation and use of the product.

Known issues with the Connection Manager Administration Kit 1.3

  • If you have a previous version of the Connection Manager Administration Kit installed on your computer, you may have to manually migrate some files. When you edit a profile created by an earlier version of CMAK, if you have made manual edits to the .INF file, these edits will have to be re-applied.
  • File names created with extended character sets (such as those containing ü, é, and other extended characters) might not be properly installed.
  • The CMAK wizard no longer provides support for automatic installation of Microsoft Dial-up Networking with your service profile. If you have Windows 95 users who need to use VPN connections they must obtain the current Dial-up Networking software by downloading it from Windows 95 (Downloads for networking and communications) at the Microsoft Windows Web site. You should document this requirement for your users. For more information on verifying and installing Dial-up Networking software, see  the "Implementing VPN support" section of the Connection Manager Administration Kit Guide.
  • All service profiles created using the CMAK wizard automatically include the browser files required by Connection Manager. If users do not have Internet Explorer installed on their computers or have an earlier version of Internet Explorer, these files are installed on their computers by the Connection Manager installer when they install the service profile. However, these files will not be installed if they conflict with the version of Internet Explorer already installed.
  • The CMAK wizard no longer provides support for automatic incorporation of an animation in your service profile. You can continue to use Macromedia animations if your platform supports this type of graphic.

If you have animations in any service profiles created using previous versions of Connection Manager, the animation settings are preserved but, after upgrading to Connection Manager 1.3, you must do the following additional steps to retain your animations in your service profiles:

    1. Obtain the self-extracting installer from the Macromedia Web site by downloading Shockwave and saving it to disk.

 

                               Be sure to obtain an appropriate license for redistribution.
 
    1. Edit the ServiceProfileFileName.inf file to include the following line under [RunPostSetupCommandsSection]:

 

                               MacromediaFlashInstaller.exe/options/flags/
 
 
               The MacromediaFlashInstaller.exe name should be a maximum of 8 characters.
 
    1. Run the CMAK wizard and add the animation file (movie) and the MacromediaFlashInstaller.exe file as additional files.

If you want to add an animation to a new service profile:

    1. Create a Macromedia animation.
    2. Create your service profile.
    3. Edit the ServiceProfileServiceName.cms file to include the following keys (or equivalent information appropriate to your animation): 
 
[Connection Manager]
AnimatedLogo=1
[Animated Logo]
Movie=Location\ServiceProfileFileName.FileExtension
Parameter
[Animation Actions]

See the Connection Manager Administration Kit Guide for more information on editing service profile files, including the parameter for the movie and information on how to specify animation actions to support a sequence of frames.

    1. Do all of the steps for preserving animations in an existing service profile, as listed above.
  • If you add a program under the [RunPostSetupCommandsSection] of the .inf file, remember to add the file as an additional file (using the CMAK wizard). Verify the complete path of each program not included as an additional file by testing the service profile on all target platforms.
  • If you had a previous version of the Connection Manager Administration Kit, please note that the location of the Profiles folder has changed from \Program Files\CMAK\Profiles-32 to \Program Files\CMAK\Profiles.
  • If you build a new .ins file, make sure all of your clients are using the the version of Connection Manager that shipped with your version of the Connection Manager Administration Kit.

Known issues with Connection Manager 1.3

For the known issues with Connection Manager 1.3, see Readme.txt (in the \Program Files\CMAK\Support\ folder). The Readme.txt file is distributed with your service profile only when you specify in the CMAK wizard that the Connection Manager software is to be included in the service profile. Edit the Readme.txt file as appropriate before building your service profile. For example, you may want to edit the Connection Manager system requirements (to reflect the total size of your service profile) and the user procedures (templates for which are found in the Connection Manager Administration Kit Guide).

For more information

For help in using the Connection Manager Administration Kit, read the Connection Manager Administration Kit Guide and related documentation.
