Breakdown of RCML components.
  1. RCML Schema:
    needs to be re-usable - elements we define here should be suitable for cross company use, e.g. the localization teams TEXT element, and controls STYLE element.
    extensible leverages name spaces - extensibility is a must - extensible from ISVs perspective so they can add additional elements into the tree - by extending the resource loader.
  2. RCML Generator
    A tool which converts existing dialog resources into RCML files. This tool generates 'standard' and 'enhanced' RCML. Enhance RCML files make use of relative positioning etc.
  3. RCML Renderer
    There are 2 parts of the renderer : the parser and the improved DialogProc . The parser builds a tree of the controls required as they appear in the RCML file. This is then converted into a dialog template. When the dialog is displayed, the new DialogProc 'walks' the controls' HWNDs and adds a new property to them which points to their XML control. These controls expose additional information, like the style of the control, which are not supported directly by USER. By accessing this information, the controls are rendered in the correct style, and any initial / default settings are applied to the control, e.g. the upper and lower bounds of a slider.
    There are 5 new API for displaying the following:
    • DialogBox -> RCMLDialog
    • PropertySheet -> RCMLPropertySheet
    • Wizard -> RCMLWizard
    • Entity Substitution -> RCMLDialogBoxTable
    • RCMLDialogBoxParam
