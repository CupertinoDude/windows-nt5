RCML  .   the product .  Status

 

Status Ending: 2 July 99

Start to implement the new extended schema, and integrate the generator into the DLL - Work is going well here, but we'll not integrate into the DLL for M0.

Work on M0 deliverables : clipping text, and resizable property sheets (re-architecture required) - on track for M0.

Work on Slide deck for first 'big' review - first pass on review done, we'll need an hour, so review moved, date TBD.

Start to convert the RCML schema to use the "XML Schema schema". A more formal schema definition, big customer win.

Status Ending: 25 June 99

Big news is that the Exchange Server (Peter Beinhardt) team loved RCML, and they said they'd sign up to be a customer. They asked hard questions, and we had the answers. They'll even supply a dev to help with the migration!

New schema implementation in design stages..

Met with CWD (consumer windows division) to extol the virtues of using GML. Good meeting, they seemed more interested in the features of RCML than GML. JeffBog was in attendance, and said Neptune would probably have something different to RCML.

Substitutable text is in (entity replacement). Has partial blessing from the XML team, but they're OK given our customer set.

Work on Slide deck for first 'big' review is progressing.

Started work on converting the RCML schema doc to use the XML schema schema with XSL for rendering - takes a while.

Did some memory investigation work, nothing looked TOO scary, we're cost the application about 80K or so.

Focussing on M0 (July 8th) deliverables.

Status Ending: 18 June 99

Last Week:

Produced concrete dates for code deliverables, M0 July 8th. http://rcml/scheduleupdate.htm

Improved the clipping text (single and multi line) with support for bigger, smaller font sizes for MSAA.

Complete removal of CString with LPCTSTR done and MSVCRT linkage.

Perf numbers, we continue to clear out the low hanging fruit before we publish numbers.

Last GML and Text+ meetings were a success, these two teams will own the localizable strings in RCML.

Had our first pure C customer use our stuff (JulieB), produced good feedback which we are rolling into our generator and SDK.
