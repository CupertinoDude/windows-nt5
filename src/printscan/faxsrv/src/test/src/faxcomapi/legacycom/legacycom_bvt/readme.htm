BVT for legacy COM API

Basic Verification Test for COM API of Whistler and Win2000

Automatic send files, and compare source with file that was received

This program will test fax sending over COM API for Win2000 Fax and Whistler Fax A .BVT file will contain Test Cases. Each Test Case is a list of Source File, Reference File, and Source Cover Page. This program will send the source file with/without the source cover page, then it will poll the modems, and then it will compare the files in the Archive (outbound), and Store In Folder of the routing method (Store) directories, with the reference files. If the reference file is the same as the files that where created and send by the fax, the BVT Passed.

 

Files needed for running:

bvtcom.bvt (configuration file)

vbtestit.exe (tiff compare utility)

tifftools.dll (DLL for vbtestit)

More files that might be required are: fxsapi.dll, various OCX/DLL from Visual Basic (all are available with Visual Basic itself)

 

 

Quick Running:
  • SETTINGS | FAX SETTINGS | RECIPIANT FAX NUMBER <- set to receive modem
  • SETTINGS | FAX SETTINGS | OUTBOUND PATH <- set to full path of any directory you wish (recommended: \outbound under BVT root)
  • SETTINGS | CASES SETTINGS | COVER PAGE <- set all cover pages to full path of cover page

Click RUN on the main GUI

Command Line Run: bvtcom {bvtcom config file}
example: bvtcom bvtcom.bvt
